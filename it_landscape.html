Progressive Web App
Core Field Application
The main field tool used by reps on site. Installable to Android and iOS home screens directly from the browser — no app store required. Runs fully offline using a service worker and localStorage cache. All pump capture, photo slots, GPS, and opportunity tracking live here.

All app files (index.html, admin.html, install.html, sw.js, and JavaScript bundles) are hosted as static files via GitHub Pages. Zero infrastructure to manage, zero cost. Deployments happen by committing to the main branch. Globally distributed via GitHub's CDN.

Service Worker
Offline Cache Manager
sw.js pre-caches all app assets on first load. Serves the app instantly from cache on subsequent visits — even with no internet. Cache is versioned (currently v38) so updates force a fresh fetch of all files. Critical for field use where connectivity is unreliable.

Backend Layer
🗄️
Supabase PostgreSQL
Primary Database
Stores all structured survey data: organisations, user profiles, audits, pumps (including photo thumbnails), and opportunities. Five tables with full Row Level Security — distributors see only their organisation's data, Schurco SA reps see all SA data, admins see everything including soft-deleted records.
PostgreSQLRow Level Security5 tables
supabase.com
🔐
Supabase Auth
Authentication & User Management
Email/password authentication with JWT sessions persisted on device. Admin creates accounts with temporary passwords via the dashboard. First-login flow forces password change and profile completion. JWT app_metadata carries the admin flag. Email confirmations disabled for immediate access.
JWT sessionsAdmin flagNo email confirm
supabase.com/docs/guides/auth
⚡
Supabase Edge Functions
SharePoint Upload Proxy
Deno-based serverless function (sharepoint-upload) acts as a secure middleman for SharePoint uploads. Holds Microsoft client credentials as environment secrets — they never touch the app code or browser. Receives Excel + photos from the app, authenticates with Microsoft Graph, creates the SharePoint folder, and uploads files. Two modes: create (folder + Excel) and photo (single image upload).
Deno runtimeSecrets-basedTwo-mode API

SharePoint Online
Photo & Excel File Storage
Permanent archive for all field survey files. Each survey creates a dedicated folder under /Shared Documents/Schurco Site Data Capture/ named {Site}_{Customer}_{Date}/. Contains the formatted Excel workbook and all pump photos named {PumpTag}_{SlotName}.jpg. SharePoint version history provides rollback on overwritten photos.

Microsoft Graph API
SharePoint Integration Layer
REST API used by the Edge Function to authenticate, locate the SharePoint document library, create folders, and upload files via PUT requests. Requires Files.ReadWrite.All and Sites.ReadWrite.All application permissions. All calls are made server-side from the Edge Function using the bearer token obtained from Entra ID.

Data Flow
How data moves through the system
📱→☁
Survey captured on site — Rep saves pump data locally to localStorage. If online, app auto-syncs to Supabase PostgreSQL immediately. Photo thumbnails (240×180px) stored in the pumps table. Full photos stay on device only.
☁→📱
New device or fresh install — On login, app pulls a lightweight manifest of all surveys from Supabase. Cloud-only surveys show as grey ☁ tiles. Tapping downloads the full record. Thumbnails load from Supabase immediately.
📱→📁
Share to SharePoint — Rep taps "Save to SharePoint". App generates Excel, sends to Supabase Edge Function. Function authenticates with Entra ID, creates folder in SharePoint, uploads Excel + all photos individually. App marks survey as SharePoint-synced.
🔐→📱
First login — Admin creates account via dashboard with temp password. Rep logs in, forced to change password via Supabase Auth, completes profile. JWT stored on device with admin flag. Session persists until explicit sign out.
🗑→☁
Soft delete by admin — Admin deletes a survey from the dashboard. Supabase sets deleted_at + deleted_by. Survey disappears from all rep views on next sync. Remains visible in admin "Deleted Surveys" section with full restore capability.
📸→☁→📁
Photo lifecycle — Captured at max 1024px JPEG 0.8 (~200KB). Thumbnail generated at 240×180 JPEG 0.6 (~10KB) and synced to Supabase. Full photo stays local until SharePoint share. Other devices see thumbnail with "Retake" prompt.
