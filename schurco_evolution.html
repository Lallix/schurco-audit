Phase Two

Multi-User Cloud Platform

Supabase backend, full authentication, organisation isolation, and an admin management portal.

v12–v15
Supabase Integration
Full multi-user database with organisations, profiles, audits, pumps, and opportunities. Login screen with email/password and Remember Me. Supabase JS bundled locally for offline auth. Organisation-level data isolation — distributors see only their own data. Admin flag in JWT app_metadata.
Supabase backendLogin screen Session persistenceOrg isolation Background syncJWT admin flag
v16–v18
Form Improvements & Bug Fixes
Contact Position changed to dropdown (12 roles). Contact Email and Contact No. added to audit form. Rep and Owner fields removed — pulled automatically from profile. CSS fix for email/tel inputs. Sync retries profile load before failing. Multiple drawer-close bugs fixed.
Contact Email + No.Position dropdown Profile auto-fillBug fixes
v19–v21
Admin Dashboard
Separate admin.html portal. Overview with KPIs and pipeline, Activity feed showing all rep submissions, Users tab with account creation, Organisations tab for distributor management. GRANT SQL fix resolved all permission issues — root cause of the earlier RLS problems.
admin.html portalKPI overview Activity feedUser creation Org managementGRANT fix

Polish, Onboarding & Data Management
Auto-sync fires after every save when online. First-login setup screen — forced password change then profile completion before app access. Photo compression (max 1024px JPEG 0.8) fixed the save-with-photos bug. Edit/delete for users and orgs in admin. Password reset button sends email. Rep initials in audit list (KvN particle handling). Delete buttons on audits and pumps.

Phase Three

Intelligence, Data Safety & Device Control

Admin map with live filtering, a three-tier data protection model, and rep-controlled device storage — all built while waiting for Microsoft OAuth approval.

v27
Admin Map + Filters + Soft Delete Architecture
Leaflet map added to admin Overview tab showing all GPS-tagged survey sites, colour-coded by best opportunity stage. Dual filter bar — Opportunity Stage (tier 1) then Representative (tier 2) — updates KPIs, pipeline, surveys table, and map simultaneously. Surveys table made expandable — clicking a row reveals all pumps with condition badges, linked opportunities, and site contact details.

Soft delete introduced across audits, pumps, and opportunities. Admin deletes flag records with deleted_at + deleted_by — invisible to reps but fully restorable. Deleted Surveys and Deleted Opportunities sections in admin show who deleted what and when, with Restore and Permanently Remove controls. RLS policies updated — only admins can delete from Supabase.
Admin mapDual filters Expandable rowsSoft delete Restore capabilityRLS hardened Deleted audit trail
v27–v28
☁/✓ Manifest Sync Model + Clear from Device
Supabase becomes the source of truth. Sync now pulls a lightweight manifest of all survey headers first — every survey is visible in the list whether locally cached (✓) or cloud-only (☁). Tapping a ☁ survey downloads the full record on demand. The 🗑 delete buttons in the field app are replaced with ☁ "Clear from device" — removes only the local cache, data stays permanently in Supabase. Admin is the only party who can truly delete records.
Manifest sync☁/✓ indicators Download on demandClear from device Supabase = source of truth
v29
Archive / Hide Function
Reps can archive surveys and opportunities to declutter their view without deleting anything. Archived records are hidden by default — a collapsible "▸ Archived (n)" toggle reveals them greyed out below active records. Unarchive with one tap. Local device only — each rep manages their own view. No Supabase changes. Admin still sees all records regardless.
Archive surveysArchive opportunities Local-only preferenceToggle to reveal No data loss
